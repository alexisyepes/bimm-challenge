Project: Car Inventory Manager - Production Ready

Target Stack: 
- React 19, TypeScript, Material UI (MUI).
- Apollo Client for GraphQL.
- MSW for API Mocking (Ensure field names match src/mocks/data.ts).

1. Data Architecture (NON-NEGOTIABLE):
   - Schema: { make: string, model: string, year: number, color: string, image: { mobile: string, tablet: string, desktop: string } }
   - Note: The 'image' field is a NESTED OBJECT. Never use flat fields for images.

2. State Management (Lifting State Up):
   - App.tsx MUST manage all shared state using React.useState.
   - States: 
     * const [searchTerm, setSearchTerm] = useState<string>("");
     * const [sortBy, setSortBy] = useState<"year" | "make">("make");
   - Child components (SearchBar, SortActions) MUST NOT have internal state for these values. They must receive value and setter via props.

3. GraphQL & Apollo Configuration:
   - Client: Use ONLY the existing client at src/graphql/client.ts.
   - Imports: 
     * In App.tsx: import { client } from './graphql/client';
     * In components: import { client } from '../graphql/client';
   - Mutations: Use 'refetchQueries: ["GetCars"]' (or matching query name) for sync.

4. Component & UI Requirements:
   - AddCarForm: A MUI Dialog (Modal) that handles its own visibility state but executes the mutation.
   - CarCard: Responsive logic for image switching at 640px and 1024px.
   - Layout: Clean, centered MUI Container.

5. Testing Requirement:
   - Create unit tests in src/components/__tests__ for CarCard and AddCarForm.
   - Verify responsive image rendering and form submission.
